# Roadmap

## Phase 1: Kernintegration

- DWF-Adapter implementieren
- Device Manager mit Locking
- Status-Endpunkt
- Scope-Capture
- Logic-Capture
- Kennwertberechnung
- OpenAPI-Schema veröffentlichen

## Phase 2: Agentenfreundliche Funktionen

- `measure/basic`
- PWM-Analyse
- Spannungslevel-Prüfung
- digitale Aktivitätsdetektion
- Claude-Code-Skill produktiv nutzbar machen

## Phase 3: Stimulus

- Wavegen-Endpunkte
- Static-I/O-Endpunkte
- konservative Guardrails

## Phase 4: Erweiterte Plattformfunktionen

- Web-Portal-Ansicht
- Messhistorie
- CSV/JSON-Export
- kombinierte Abläufe mit GPIO, Serial und Digilent

## Phase 5: Erweiterte Testorchestrierung

- zusammengesetzte ESP32-Testpläne
- Trigger-Capture gekoppelt mit Pi-GPIO-Reset
- optionale Protokoll-Dekodierung
